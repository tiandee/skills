# 简历内容优化

## 原则

- **事实优先**：个人项目不称「生产级/企业级」；职称与真实职位一致；倍数/数据需可解释。
- **措辞降级**：经验不足 1 年的领域用「深入实践/深入使用」替代「精通」；避免「国内早期」「弥合鸿沟」等无法证明的表述。
- **结构清晰**：教育背景、个人评价等按目标岗位习惯放置；核心优势与项目描述前后呼应。

## 去夸大检查清单

| 类型 | 易夸大表述 | 建议改法 |
|------|------------|----------|
| 项目性质 | 生产级、企业级（实为个人项目） | 完整/全栈/工程化、或删除定语 |
| 能力词 | 精通（年限或项目数不足） | 深入实践、深入使用、熟练使用 |
| 身份 | 核心架构师（实际职称非此） | 核心技术骨干、技术专家等与 JD/职级一致 |
| 影响力 | 国内早期、弥合鸿沟、10 倍效率（无依据） | 删除或改为「显著提升」「注重…实践」 |
| 数据 | 推理效率提升 3 倍（易被追问定义） | 直接写「步数从 26 降至 8」等可验证数据 |
| 落地 | 均有项目落地、非 Demo 层面（个人项目） | 均有完整项目实践，覆盖设计/开发/评测全流程 |

## 优化流程

1. **读当前简历**（MD 或指定路径），明确目标岗位/JD（若有）。
2. **JD 对齐**（若提供 JD）：提取 Must-have/Nice-to-have 关键词；在核心优势、技能、项目描述中显性呼应；调整 bullet 顺序，把最相关置顶。
3. **事实核查**：逐段对照「去夸大检查清单」；个人项目一律去掉生产级/企业级；职称与简历标题/经历一致；倍数改为可解释的原始数据。
4. **结构微调**：教育背景、个人评价位置按行业习惯；个人评价用关键词罗列（如 LangGraph 状态机、记忆机制、Supervisor 任务分解）+ 一句业务结合（如「能与业务场景有效结合」）。
5. **输出**：修改后的全文或 diff 说明；若只做建议，则给出「原文 → 建议」列表并注明理由。

## 写作规范

- 中文为主，技术术语保留英文（LangGraph、RAG、Trace ID 等）。
- Bullet 格式：**技术关键词** + 动作 + 具体做法 + 量化/结果。
- 禁止虚构数据；无法精确时用「约」「显著」等保守表述。
