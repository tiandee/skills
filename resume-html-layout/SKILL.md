# 简历 HTML 排版

## 依赖技能

执行本技能时，应**优先使用或对齐** [Anthropic Skills - frontend-design](https://github.com/anthropics/skills/tree/main/frontend-design) 的设计规范与产出质量：配色、字体、布局原则、避免通用 AI 审美等。简历 HTML 为前端视觉产物，生成单文件 HTML 时遵循 frontend-design 的「高设计质量、非通用模板」要求；本技能在此基础上补充**双页 A4、打印、分页**等简历特有约束。

## 目标

- 从已有 **Markdown 简历** 生成 **单文件 HTML**，便于浏览器打开与「另存为 PDF」。
- 双栏布局：左侧窄栏（个人信息/教育/技能/个人评价），右侧主栏（核心优势/项目/经历）。
- **严格两页 A4**：内容不溢出第三页；第二页顶部留白，避免贴边；关键块（如个人评价）不被截断到两页。

## 结构约定

- **第一页**：侧栏（姓名、定位、联系方式、教育、技能）；主栏（核心优势、开源贡献、第一个 AI 项目或前半内容）。
- **第二页**：侧栏（个人评价等延续或单独成块）；主栏（后续项目、工作经历）。
- 若内容易溢出，给第二页使用**紧凑样式类**（如 `.page-2`）：缩小 gap、margin、列表行距，保证底部内容完整露出。

## 技术要点

### 分页与高度

- 每页一个 `.page` 容器，`height: 297mm`（A4 高），`overflow: hidden`，避免单页内容撑破高度。
- 打印时：`@media print` 内 `page-break-after: always`（最后一页用 `auto`）；`@page { size: A4; margin: 0; }`。

### 打印样式

- 侧栏深色在打印时易变灰。必须加：
  - `* { -webkit-print-color-adjust: exact; print-color-adjust: exact; }`
  - `.sidebar { background-color: #111827 !important; }`（及其它需保留的背景色）。
- 提醒用户：打印时勾选「背景图形」。

### 两页内容分配

- 若「个人评价」在侧栏且被分页截断，可将个人评价整块移到**第二页**的侧栏顶部，并加 `padding-top` 作留白。
- 第二页主栏同样加适量 `padding-top`，避免标题贴顶。

### 样式建议

- 字体：Noto Sans SC + 系统无衬线；正文字号 8.5–9.5pt，行高 1.5–1.6。
- 侧栏：深色背景（如 #111827），浅色字；区块标题用 accent 色（如 #06b6d4）下划线或左边框。
- 主栏：白底；章节标题用底边线；要点用左侧色条 + 浅底。
- 奖项/标签：统一用 `.award-badge` 等类，打印时在 `@media print` 里显式指定背景色。

## 操作流程

1. **读取**用户指定的简历 Markdown（或默认路径）。
2. **解析**为语义块：个人信息、教育、技能、核心优势、开源、项目、工作经历、个人评价。
3. **生成 HTML**：按上述双栏与两页结构写入；CSS 内联或同一文件内 `<style>`，包含常规样式与 `@media print`。
4. **两页校验**：若存在「第三页有少量内容」或「第二页底部被裁」，则收紧第二页的 padding/gap/margin（或为第二页加 compact 类），必要时微调字号/行距，直到两页刚好填满且无溢出。
5. **交付**：提供单文件 HTML 路径；简要说明「用浏览器打开后 Ctrl+P 选保存为 PDF，并勾选背景图形」。
